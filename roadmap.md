# FieldCore Roadmap

## Phase 7: Notifications Engine

Add proper notifications for key business events:

- New bookings
- Quote sent
- Quote accepted/rejected
- Invoice sent
- Payment received
- Job scheduled/rescheduled
- Worker assigned
- Job completed

### Channels

- Email first
- WhatsApp later
- SMS optional

### Rule

Do not overbuild. Start with email notifications first.

---

## Phase 8: Public Booking / Request Portal

This is where businesses' customers can submit service requests without logging in.

Add:

- Public booking form
- Service selection
- Property details
- Photo uploads
- Preferred date/time
- Request tracking
- Admin approve/reject/quote flow

This turns FieldCore into something companies can actually use with customers.

---

## Phase 9: Proof-of-Work System

Add stronger job completion proof.

Workers should be able to upload/add:

- Before photos
- After photos
- Notes
- Client signature
- Completion timestamp
- GPS/location if needed

This is important for trust, disputes, and receipts.

---

## Phase 10: Production Readiness

Before trying to sell this seriously, add:

- Backup plan
- Better error logs
- Rate limiting
- Audit log review UI
- Seed/demo reset command
- Deployment checklist
- Stronger environment variable checks
- Production security review

---

## Phase 11: SaaS Billing / Subscriptions

This is where FieldCore becomes a real SaaS product.

Add:

- Plans
- Company subscription status
- Trial period
- Usage limits
- Locked features by plan
- Billing page
- Payment provider integration

---

## Phase 12: Reporting & Analytics

Add reports for owners/admins:

- Revenue
- Unpaid invoices
- Completed jobs
- Worker performance
- Service popularity
- Quote conversion
- Customer history
